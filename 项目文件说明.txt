================================================================================
                        项目文件结构说明
================================================================================

📁 主要目录
================================================================================

backed/                     # 后端代码（Java + Spring Boot）
fronted/                    # 前端代码（Vue 3 + TypeScript）
  ├── .env.production       # 生产环境配置 ✅
  ├── .env.development      # 开发环境配置 ✅
  └── dist/                 # 打包后的文件（需上传）
deploy/                     # 部署脚本（上传到服务器用）
  ├── start.sh              # 启动脚本
  ├── stop.sh               # 停止脚本
  ├── restart.sh            # 重启脚本
  ├── check.sh              # 环境检查脚本
  └── nginx.conf            # Nginx配置示例

📄 说明文档（都在根目录）
================================================================================

✅ 开始部署.txt            # 最简洁的部署步骤
🔥 重新部署前端.txt        # 前端重新打包上传指南 ⭐
部署说明.md                # 完整详细的部署文档
README.md                  # 项目整体说明
打包部署.bat               # Windows打包脚本

📦 打包流程
================================================================================

双击运行：打包部署.bat

选项：
[1] 打包后端 - 生成 backed/target/knowledge-management-system-1.0.0.jar
[2] 打包前端 - 生成 fronted/dist/ 下的所有文件
[3] 打包全部 - 同时打包前后端

📤 上传到服务器
================================================================================

后端文件 → /www/wwwroot/knowledge-backend/
  - backed/target/knowledge-management-system-1.0.0.jar
  - deploy/start.sh
  - deploy/stop.sh
  - deploy/restart.sh
  - deploy/check.sh

前端文件 → /www/wwwroot/www.personalmanange.xyz/
  - fronted/dist/ 里面的所有文件

⚙️ 服务器常用命令
================================================================================

启动后端：
  cd /www/wwwroot/knowledge-backend
  ./start.sh

停止后端：
  ./stop.sh

重启后端：
  ./restart.sh

查看日志：
  tail -f logs/application.log

检查环境：
  ./check.sh

🔧 环境变量说明
================================================================================

fronted/.env.production（生产环境）：
  VITE_API_BASE_URL=http://www.personalmanange.xyz/api

fronted/.env.development（开发环境）：
  VITE_API_BASE_URL=http://localhost:8080/api

⚠️ 每次修改环境变量后，需要重新打包前端！
